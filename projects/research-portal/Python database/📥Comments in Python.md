- This is the common practice for multi-line comments in Python, even though they're technically not treated as comments by the interpreter.
- Comments are indeed used for documentation purposes and do not affect the execution of the code.

# Detailed Notes:


Comments can be used to explain the code for more readability. Comments are a crucial part of programming in Python (as in most programming languages), serving several important purposes:

1. **Explanation**: They can explain what a particular part of the code does, making it easier for someone (including your future self) to understand the code when they look at it later.
    
2. **Debugging**: Sometimes, you might want to temporarily disable a piece of code without deleting it. You can turn these lines into comments, which Python will ignore during execution.
    
3. **Readability**: Comments can be used to break up the code into sections, making it more readable and organized.
    

In Python, there are two types of comments:

### Single-line Comments:

- Begin with a hash symbol (`#`).
- Everything following the `#` on that line will be ignored by Python.

Example: 

```python
# This is a single-line comment
print("Hello, World!")  # This prints a greeting

```

### Multi-line Comments:

Python doesn't have a distinct syntax for multi-line comments. However, you can use a multi-line string (enclosed in triple quotes) as a comment. While not technically a comment, Python will ignore these strings if they're not assigned to a variable.

Example:

```python
"""
This is a multi-line comment
It spans multiple lines
"""
print("Hello, World!")

```
